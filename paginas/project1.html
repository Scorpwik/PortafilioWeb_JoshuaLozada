El Reto & La Solución

El objetivo principal era crear un sistema interactivo robusto que no dependiera de motores gráficos preconstruidos. Para lograr la fluidez del juego y el manejo de múltiples asteroides y enemigos simultáneos, implementé un sistema basado estrictamente en Programación Orientada a Objetos (POO).

Se desarrollaron clases padre para los elementos del espacio, aplicando conceptos de herencia (child structures) para diversificar los comportamientos de los objetos sin saturar la memoria.

Mecánicas Destacadas

  • Polimorfismo: Uso de métodos compartidos para gestionar las colisiones de diferentes tipos de entidades.
  • Estructuras Child: Jerarquía de clases para optimizar la creación de niveles y enemigos.
  • Físicas Personalizadas: Algoritmos matemáticos escritos en Java para simular aceleración e inercia en gravedad cero.
  • Sistema de Puntuación: Tracking de datos en tiempo real.
